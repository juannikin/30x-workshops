Paso 1: Extraccion Estructurada

De texto crudo a datos accionables

Lo que extraemos

  1. Decisiones: que se acordo?
  2. Action items: quien hace que y para cuando?
  3. Pain points: que duele? En sus palabras.
  4. Objeciones: que los detiene?
  5. Buying signals: budget, timeline, urgency
  6. Competitor mentions: que mas evaluan?
  7. Key quotes: frases textuales para content

Output: JSON estructurado

{ "decisions": [...], "action_items": [...], "pain_points": [...], "objections": [...], "buying_signals": { "budget": "$15-20k/mo", "timeline": "Q2", "urgency": "high" }, "competitors": [...], "key_quotes": [...] }

Paso 3: Transcript → Landing Page

Copy que suena como tu ICP — porque viene de sus propias palabras

Estructura de la landing

  1. Hero: headline from top pain point
  2. Problem: 3 pain points del ICP
  3. Solution: how you solve each one
  4. Social proof: stats + testimoniales
  5. CTA: book a call / start free

Por que funciona

  • El lenguaje no es inventado — viene de calls
  • Las objeciones se pre-responden en la pagina
  • Los pain points son los reales, no los asumidos
  • Cada vertical puede tener su propia landing

5 calls con CTOs → landing para CTOs. 5 calls con VPs Marketing → landing para VPs Marketing.

Tipos de posts que salen

  • Pattern post: "Hable con 5 VPs esta semana y todos dijeron lo mismo..."

El AOP del Transcript Agent

Post-Call Routine (automatica)

  • T+0 min — ingest_transcript() desde Granola
  • T+1 min — extract_insights() → JSON estructurado
  • T+2 min — update_crm_from_call() → deal record actualizado
  • T+3 min — Notificar en Slack: resumen + action items + signals

Content Cadence (semanal)

  • Viernes PM — Batch: 5 transcripts de la semana → generate_social_posts(3)
  • Mensual — Insights acumulados → generate_landing_page() refresh
  • Trimestral — Insights acumulados → generate_ad_copy() para nuevas campanas

Escalacion: si sentiment es "at-risk" → notificar manager. Si competitive mention es nueva → update battlecard.
